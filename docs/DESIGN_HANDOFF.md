## 1. Темы и токены

**Как в проекте**

- Глобальные стили: `frontend/src/app/globals.css`.
- Tailwind v4: директива `@import "tailwindcss"`, `@theme inline { ... }` мапит **семантические** цвета на CSS-переменные (`--color-accent`, `--color-bg`, `--color-text-primary`, …).
- Базовые значения по умолчанию — в `:root` (палитра «по умолчанию» до переключения темы).
- Альтернативные визуальные режимы — классы на корне: `.theme-clinical`, `.theme-modern`, `.theme-warm`, `.theme-clinical-aura`, `.theme-editorial-luxe`, `.theme-swiss-precision` и др. Каждый переопределяет `--accent`, `--bg`, `--text-*`, `--border-*`, иногда `--font-family-heading` / `--font-family-body`.

**Активная тема в продакшн-верстке:** на `<html>` в `frontend/src/app/layout.tsx` навешан класс **`theme-clinical-aura`** (плюс переменные шрифтов от `next/font`).

**Как в `theme-clinical-aura` (фрагмент логики):**

- Акцент: розовый `#E8638B`, контраст текста на кнопках — светлый (`--accent-contrast: #ffffff`).
- Фон страницы: `#ffffff`, фон «карточек/вторичный» — `#F8F9FB`.
- Текст: иерархия `--text-primary` → `--text-secondary` → `--text-muted`.
- Границы: нейтральные серо-синие (`--border-color`, `--border-mid`).
- Дополнительный акцент темы: `--ca-secondary` (бирюзовый), для акцентных блоков на главной.

**Рекомендация:** в новом проекте заранее зафиксировать **одну** прод-тему и документировать её токены; остальные темы — как эксперименты или смена «бренда» через один класс на `<html>`.

## 2. Шрифты и активная тема

**Как в проекте**

- Шрифты подключаются через **`next/font/google`** в `frontend/src/app/layout.tsx`; каждому семейству задана CSS-переменная (`--font-playfair`, `--font-jakarta`, `--font-inter`, …).
- На `<html>` перечислены **все** переменные шрифтов, чтобы при смене класса темы не пересобирать layout.
- В **`theme-clinical-aura`** заголовки: Plus Jakarta Sans (`--font-jakarta`), основной текст: Inter (`--font-inter`).
- Глобально в `globals.css`: `body` использует `font-family: var(--font-family-body)`, заголовки `h1–h6` — `var(--font-family-heading)`.

**Рекомендация:** не полагаться на один Inter для всего; пара **display / body** даёт узнаваемость без тяжёлых кастомных шрифтов.

## 3. Компонентный слой `shared/ui`

**Как в проекте** — единая точка экспорта: `frontend/src/shared/ui/index.ts`.

| Компонент | Назначение |
|-----------|------------|
| `Button` | Кнопки, варианты (primary / secondary / ghost и т.д.) |
| `Card` | Карточки контента |
| `Badge` | Метки |
| `Input` | Поля ввода с подписью и ошибкой |
| `Select`, `DropdownSelect` | Выбор значений |
| `Modal` | Модальные окна |
| `Skeleton`, `SkeletonCard`, `SkeletonTableRow` | Состояния загрузки |
| `PaymentStatusBadge`, `ModerationStatusBadge` | Предметные бейджи |
| `EmptyState` | Пустые состояния |
| `PageLoader` | Полноэкранная загрузка |
| `ContentBlockRenderer`, `DocumentContentBlockRenderer` | Рендер блоков контента из API |

Страницы в `app/` собираются из этих примитивов + виджетов (`widgets/`) и сущностей (`entities/`).

## 4. Типографика и ритм

**Как в проекте**

- Заголовки страниц часто используют класс **`font-heading`** (мапится на `--font-heading` в `@theme`).
- Крупные секции на главной: вертикальные отступы вида **`py-20 lg:py-28`**, контент в **`mx-auto max-w-7xl px-4 lg:px-8`** — повторяемый «коридор» контента.
- Цвет текста: **`text-text-primary`**, **`text-text-secondary`**, **`text-text-muted`** вместо произвольных hex в каждом месте (где соблюдена дисциплина).

**Рекомендация:** держать **одну сетку** (`max-w-7xl` + горизонтальные отступы) для всех маркетинговых страниц — визуально стабильнее, чем пляшущие `max-w-6xl` / `max-w-5xl` без правил.

## 5. Иконки и motion

**Как в проекте**

- Иконки: **`lucide-react`** — единый стиль, масштабирование через классы Tailwind.
- Анимации: **framer-motion** (например главная), **GSAP** + обёртка `useGSAP` в `frontend/src/shared/lib/useGSAP.ts` для секций с `staggerReveal`, `parallaxY` (`frontend/src/shared/lib/animations.ts`).

**Рекомендация:** не смешивать 5 библиотек иконок; motion — осмысленно (скролл, появление секций), не «везде подёргивать».

---

## 6. Макет: шапка, подвал, сетка

**Как в проекте**

- **Header** (`frontend/src/widgets/header/Header.tsx`): `sticky top-0`, фон `bg-bg-secondary/95`, граница `border-border`, логотип + навигация + Telegram (если есть в публичных настройках) + авторизация. Контейнер: **`max-w-7xl px-4 py-3 lg:px-8`**.
- **Footer** (`frontend/src/widgets/footer/Footer.tsx`): тёмный фон **`bg-[#4a4a4a]`**, контрастные ссылки/текст; контент с тем же **`max-w-7xl`**. Под футером — блок способов оплаты (**`PaymentMethodsStrip`**) на светлом фоне (`bg-bg-secondary`) — визуальное отделение юридического/платёжного блока.
- Данные контактов в футере частично из API (`usePublicSettings`), частично захардкожены (адрес, ОГРН) — для handoff важен **визуальный** паттерн: тёмный низ + светлая полоса под ним.

## 8. Как не делать похоже на шаблон нейросети

Связано с **фактическими** решениями в репозитории (и обобщениями):

1. **Две роли типографики** — заголовки и body разведены через CSS variables и темы, а не один системный шрифт «на всё».
2. **Ограниченная палитра** — акцент, фон, три уровня текста и нейтральные бордеры; нет «радужного» набора случайных цветов по странице.
3. **Семантические токены в Tailwind** — `bg-bg`, `text-text-primary`, `border-border` вместо длинных произвольных hex в каждом компоненте (где дисциплина соблюдена).
4. **Тёмный футер + светлая основа** — якорь «официальности» и отделение служебной/юридической зоны от контента.
5. **Одна сетка контента** — `max-w-7xl` + горизонтальные отступы как повторяемый паттерн.
6. **Иконки одной библиотеки** (Lucide) — визуальная согласованность.
7. **Акценты в контенте** (главная) — локальные цвета для карточек миссии согласованы с темой (`#edbecc`, `#5BB5A2`), а не случайный градиент «как везде».
8. **Глобальные утилиты в CSS** — например `bg-metal-gradient`, `bg-noise` в `globals.css`: ограниченный набор эффектов, а не дефолтный «glassmorphism» на каждой секции.
9. **Темы как классы** — смена «настроения» сайта без переписывания всех компонентов.
10. **Формы** — общие `Input`/`Button`/`Card`, единый язык ошибок и отступов в кабинете.
11. **Не смешивать бренд футера и маркетинговую полосу** — платёжные логотипы вынесены в отдельную светлую полосу под футером (`PaymentMethodsStrip`).
12. **Доступность контраста** — акцентные кнопки в теме `clinical-aura` используют светлый текст на насыщенном фоне (`--accent-contrast`).

## 9. Чек-лист на будущие проекты

1. Зафиксировать **CSS variables** для цветов и шрифтов до массовой вёрстки страниц.
2. Выбрать **одну пару шрифтов** (heading + body) и подключить через `next/font`.
3. Ограничить **количество акцентных цветов** (1 основной + при необходимости 1 вторичный).
4. Ввести **единую контентную сетку** (`max-width` + padding) для всех публичных страниц.
5. Собрать **набор UI-примитивов** (кнопка, поле, карточка) до копирования экранов.
6. Иконки — **одна библиотека**, один размер по умолчанию в UI-kit смысле.
7. Анимации — **точечно** (hero, секции), не на каждом `div`.
8. Для «официальных» блоков (футер, документы, оплата) — **отдельный визуальный регламент** (фон, типографика).
9. Проверить **контраст** акцент/кнопка/текст для основной темы.
10. Документировать **активную тему** (класс на `<html>`) и не оставлять «мертвые» темы без согласования.
