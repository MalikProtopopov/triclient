### 1.3 Общие паттерны

**Состояния каждой страницы:**

| Состояние | Отображение                                                                |
|-----------|----------------------------------------------------------------------------|
| Loading   | Skeleton-loader, повторяющий структуру контента (не спиннер на весь экран) |
| Empty     | Иллюстрация + текст + CTA-кнопка (например, «Пока нет врачей. Импортировать из Excel?») |
| Error     | Красный alert-блок с текстом ошибки + кнопка «Повторить»                   |
| Success   | Toast (sonner) в правом верхнем углу, auto-dismiss 5 сек                   |

**Toast-уведомления (sonner):**
- Успех операции (создание, обновление, удаление) — зелёный toast
- Ошибка API — красный toast с текстом `error.message`
- Сетевая ошибка — красный toast «Ошибка соединения. Повторите попытку»

**Пагинация:** серверная, offset-based. Компонент пагинации внизу таблицы: номер страницы, стрелки, выбор per_page (20/50/100). Infinite scroll не используется — только классическая пагинация.

#### Страница входа

| Параметр    | Значение                   |
|-------------|----------------------------|
| URL         | `/admin/login`             |
| Доступ      | Неавторизованные           |
| Назначение  | Вход в админпанель         |

**Компоненты:**
- Логотип ассоциации по центру
- Заголовок: «Вход в панель управления»
- Форма логина
- Ссылка «Забыли пароль?»

**Поля формы:**

| Поле     | Тип инпута  | Обязательно | Валидация                       |
|----------|-------------|-------------|---------------------------------|
| Email    | `<Input type="email">` | Да | Формат email, max 255 символов |
| Пароль   | `<Input type="password">` | Да | Min 1 символ                  |

**Кнопки:**

| Кнопка    | Действие                        | API                              | Состояния                                  |
|-----------|----------------------------------|----------------------------------|--------------------------------------------|
| Войти     | Отправка формы                   | `POST /api/v1/auth/login`        | Default → Loading (спиннер) → Success (redirect) / Error (toast) |

**Логика:**
- При успехе: сохранить `access_token` в памяти (не localStorage), redirect на `/admin/dashboard`
- `refresh_token` приходит в HttpOnly cookie автоматически
- При 401: toast «Неверный email или пароль»
- При 403 (ACCOUNT_DEACTIVATED): toast «Аккаунт деактивирован»
- После логина проверить роли: если нет admin/manager/accountant → toast «Нет доступа к админпанели» + logout

**API-запросы:**

| Метод | Endpoint                | Когда                  |
|-------|-------------------------|------------------------|
| POST  | `/api/v1/auth/login`    | Отправка формы          |
| POST  | `/api/v1/auth/refresh`  | При старте приложения (проверка сессии) |

| Фильтр              | Тип компонента    | Значения                                           | Query param           |

| Данные изменились   | `<Select>` или чекбокс | Да / Нет (подсказка: проверять, изменились ли данные) | `has_data_changed` |

Кнопка «Сбросить фильтры» — появляется если хотя бы один фильтр активен.

**Таблица:**

| Колонка             | Поле                     | Сортировка | Ширина |

1. **Первичная заявка** — при `moderation_status === "pending_review"` и отсутствии `pending_draft`: текст «Первичная заявка (анкета и документы)».

Блок размещается над вкладками, с жёлтой рамкой для визуального выделения.

- Фото (изображение)

- Публичный email

- При выборе `external` → показать поле «URL видео», скрыть «Файл видео»

- При редактировании записи с `uploaded`: показать имя файла + размер + кнопку «Заменить файл»
- Прогресс-бар загрузки для больших файлов (> 100 MB)

**Drag-drop зона загрузки видео (react-dropzone):**
- Текст: «Перетащите видеофайл или нажмите для выбора»
- Поддерживаемые форматы: MP4, WebM, MOV

- После выбора файла: имя файла, размер, кнопка «Удалить»

- Редактор документа: секция «Контентные блоки» (только в edit mode)

## 5. Middleware и авторизация (Next.js)

### 5.1 Middleware

```typescript
// middleware.ts
// Проверяет наличие access_token:
// - Если нет и страница не /admin/login → redirect на /admin/login
// - Если есть и страница /admin/login → redirect на /admin/dashboard
// - Проверка ролей: если нет нужной роли → 403 страница
```

### 5.2 Обновление токена

- Access token хранится in-memory (React state / Zustand)
- Axios interceptor: при 401 → вызов `POST /api/v1/auth/refresh` → retry original request
- Если refresh тоже 401 → redirect на `/admin/login` с toast «Сессия истекла»

### 5.3 Выход

Кнопка «Выйти» в сайдбаре:
1. `POST /api/v1/auth/logout`
2. Очистить in-memory token
3. Redirect на `/admin/login`
