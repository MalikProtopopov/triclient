## 6. API без mock (при USE_MOCK=true идут на бэкенд)

При `NEXT_PUBLIC_USE_MOCK=true` следующие запросы **не** перехватываются и уходят на реальный API:

- Auth: login, logout, refresh, forgot-password, reset-password
- Doctors: moderate, approve-draft, toggle-active, send-email, send-reminder, import, import/{taskId}
- Events: POST, PUT, DELETE
- Articles: POST, PUT, DELETE
- Organization documents: DELETE
- Plans: PUT
- Pages SEO: PUT
- Settings: PUT
- Voting: POST, PATCH
- Users: POST, DELETE

Для полностью автономной работы без бэкенда потребуются mock-обработчики для этих эндпоинтов.
